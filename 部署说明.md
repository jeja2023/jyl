# 甲友乐 JYL 项目部署指南

该指南将指导您如何将“甲友乐”项目从开发环境部署到生产服务器（Linux/Ubuntu/CentOS）。

## 1. 部署架构说明

*   **前端**：uni-app (Vue 3) 构建的 H5产物。
*   **后端**：Node.js (Koa 2) 服务，负责提供 API 接口并直接托管前端静态文件。
*   **数据库**：MySQL 5.7+。
*   **进程管理**：PM2 (确保后端服务挂掉后自动重启)。
*   **反向代理**：Nginx (负责 SSL 证书配置及请求转发)。

---

## 2. 准备工作

在服务器上安装以下必要软件：
- **Node.js**: v16 或更高版本。
- **MySQL**: 5.7+。
- **PM2**: `npm install -g pm2`。
- **Git**: 用于拉取代码。

---

## 3. 部署步骤

### 第一步：获取代码并安装依赖

```bash
# 克隆代码
git clone <your-repository-url>
cd jyl

# 安装后端依赖
cd server
npm install

# 安装前端依赖
cd ../client
npm install
```

### 第二步：配置环境并准备前端产物

1.  **前端构建**：
    在 `client` 目录下执行构建命令，生成 H5 静态资源：
    ```bash
    npm run build:h5
    ```
    构建产物将位于 `client/dist/build/h5`。

2.  **后端环境配置**：
    进入 `server` 目录，将 `.env.example` 复制为 `.env` 并根据实际情况修改：
    ```bash
    cp .env.example .env
    # 使用 vim 或 nano 编辑配置数据库连接、JWT 密钥、腾讯云 OCR 密钥等
    vim .env
    ```

3.  **数据库初始化**：
    确保服务器上已创建对应的数据库名，后端通过 `DbService.init()` 在启动时会自动同步表结构并初始化种子数据（百科条目、贴士等）。

4.  **数据库迁移（生产环境）**：
    生产环境已有数据时，不建议使用自动 `sync alter`。请使用迁移脚本进行结构升级：
    ```bash
    cd server
    npm install
    # 确保已设置环境变量：DB_HOST / DB_PORT / DB_NAME / DB_USER / DB_PASS
    npm run migrate
    ```
    如果服务器设置了 `NODE_ENV=production`，请使用 `npm install --include=dev` 以安装 `sequelize-cli`，
    或改用 `npx sequelize-cli db:migrate`。
    如需回滚最近一次迁移：
    ```bash
    npm run migrate:undo
    ```
    迁移说明文档位于：`server/migrations/README.md`。

5.  **Docker 部署执行迁移**：
    若使用 Docker 部署，迁移需要在容器内执行：
    ```bash
    cd docker
    # 确保容器已启动
    sudo docker compose --env-file .env_docker exec jyl-app npm run migrate
    ```
    如需回滚最近一次迁移：
    ```bash
    cd docker
    sudo docker compose --env-file .env_docker exec jyl-app npm run migrate:undo
    ```
    旧版命令可将 `docker compose` 替换为 `docker-compose`。

    建议在 `.env_docker` 中显式设置 `NODE_ENV=production` 与 `DB_SYNC_ALTER=false`，避免生产环境自动改表。

### 第三步：使用 PM2 启动后端服务

在 `server` 目录下启动：
```bash
pm2 start index.js --name jyl-server
pm2 save
pm2 startup
```
*提示：后端会自动托管上一步生成的 `client/dist/build/h5` 中的静态文件，您无需单独配置 Nginx 托管前端。*

## 4. (可选) 配置 Nginx 与 SSL

通常建议在前端增加一层 Nginx 反向代理，以提供 HTTPS 支持：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    location / {
        proxy_pass http://localhost:3000; # 转发到 Koa 后端
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 5. 存储与权限

确保后端进程对根目录下的 `storage` 文件夹有读写权限，以便能够保存 OCR 上传的化验单图片：
```bash
sudo chown -R $USER:$USER storage
chmod -R 755 storage
```

## 6. 常见问题排查

1.  **页面 404**：如果您直接在浏览器地址栏刷新 SPA 路由（如 `/pages/record/list`），后端已经配置了回退机制返回 `index.html`。
2.  **图片不显示**：检查 `server/.env` 中的环境变量，确保访问地址与后端端口或 Nginx 配置一致。
3.  **数据库连接错误**：请确保 MySQL 运行正常，且 `.env` 中的用户名、密码 and 端口填写正确。

## 7. 使用 Docker 一键部署 (推荐方案)

为了保持项目根目录整洁，所有 Docker 相关文件均存放在专用的 **`docker/`** 目录下。

### 7.1 全新部署 (Fresh Install)

1.  **准备环境**：
    确保服务器已安装 **Docker** 和 **Docker Compose**。

2.  **配置环境参数**：
    在 `docker/` 目录下，将 **`.env_docker.example`** 复制为 **`.env_docker`**：
    ```bash
    cd docker
    cp .env_docker.example .env_docker
    # 使用 vim 编辑业务配置 (如 JWT 密钥、OCR 的 SecretID/Key 等)
    vim .env_docker
    ```

3.  **构建并启动容器**：
    在 `docker/` 目录下执行：
    ```bash
    # 使用 --build 强制从源码重新编译前端和后端镜像
    sudo docker compose --env-file .env_docker up -d --build
    ```
    *注：首次启动可能需要几分钟时间来下载 MySQL 指向和安装 NPM 依赖。*

### 7.2 升级部署 (Update & Redeploy)

当您拉取了新的代码（例如路由变更、指标名称修正）后，请按以下步骤操作：

1.  **同步最新代码**：
    ```bash
    git pull origin main
    ```

2.  **一键重构并更新**：
    在 `docker/` 目录下执行，这将触发多阶段构建，**自动清理旧的 H5 缓存文件**：
    ```bash
    sudo docker compose --env-file .env_docker up -d --build jyl-app
    ```

3.  **验证版本**：
    访问您的域名，进入“我的”页面，检查底部的版本号（如 v1.6.7）是否已更新。

### 7.3 💡 生产环境避坑指南 (PWA & Cache)

*   **PWA 缓存问题**：如果您部署在美国服务器且网络延迟高，浏览器可能会缓存旧的 `sw.js`。代码更新后，建议在开发者模式下的 `Application -> Storage` 点击 **Clear site data**，或按 `Ctrl+F5` 强制刷新。
*   **API 地址注入**：对于多端部署（如未来发布小程序），可以在构建时注入域名：
    ```bash
    # 仅在非同源部署时需要
    sudo docker compose build --build-arg VITE_API_BASE=https://api.yourdomain.com jyl-app
    ```

### Docker 方案优势

- **物理清理**：项目已配置 `emptyOutDir: true`，Docker 每次构建都会清空镜像内的 `dist/assets` 目录，绝对不会残留旧版本的 JS 代码碎片。
- **环境隔离**：Node.js 与 MySQL 版本完全独立，不受宿主机干扰。
- **持久化挂载**：`storage` 图片目录和 `mysql_data` 数据库卷已自动映射，重启后数据不丢失。
- **自动化反代**：Node.js 后端自动托管构建好的 H5 静态文件，无需额外配置 Nginx 映射静态路径。
