# 鏇存柊鏃ュ織 (ChangeLog)
## [1.8.0] - 2026-05-19

### Android APK 正式打包链路
- 完成 Android APK 首包发布准备，版本统一升级为 `1.8.0`，`versionCode` 升级为 `180`。
- 配置 DCloud AppID、正式包名、正式自有签名证书打包流程，支持 HBuilderX 云打包与 Linux HBuilderX CLI 打包路径。
- App 构建生产 API 地址固定为 `https://jyl.880301.xyz`，避免 APK 内退回 `localhost:3000` 导致手机无法联网。
- Android ABI 明确限制为 `arm64-v8a`，避免 x86 兼容提示和无效包体膨胀。
- Android 权限按最小权限原则收紧，移除读日志、读取账号、读取手机状态、写系统设置等无关高风险权限。

### 一次安装与后续热更新
- 新增 App 启动热更新检查机制，仅在 `APP-PLUS` 生效，不影响 H5 和小程序。
- 新增 `/api/app/update/check` 接口，后端读取 `storage/app-updates/manifest.json` 下发最新 wgt 包信息。
- 新增 `npm run release:app`，一键生成 App 离线资源与 wgt 热更新包。
- 新增 `npm run app:update:publish`，将 wgt 发布到 `storage/app-updates/`，自动计算 size、md5、sha256，并阻止 `versionCode` 回退或重复发布。
- 登录页新增 H5 专用“下载安卓版”入口，默认指向 `/storage/app-releases/jyl.apk`，方便用户首次安装 APK。
- 支持普通更新、强制更新、最低可用版本门槛和 release notes；用户下载更新时展示进度，安装完成后自动重启。
- 下载地址要求 HTTPS 或与 API 服务同源，降低热更新包分发风险。

### 图标与打包稳定性
- 修复 Android 图标配置错误：原 `logo.png` 实际为 ICO 内容，导致 HBuilderX 校验 `app-plus.distribute.icons.android.*` 失败。
- 新增 `npm run icons:app`，从 `logo.ico` 生成标准 Android PNG 图标：hdpi、xhdpi、xxhdpi、xxxhdpi。
- 移除中文文件名静态资源 `甲友乐.ico`，避免 HBuilderX App 打包时报“文件名不能包含中文或全角字符”。
- 登录页和关于页 Logo 改用 `/static/app-icons/icon-xxxhdpi.png`，避免 App 端继续引用 ICO。

### 发布文档与安全清单
- 新增 `ANDROID_APK_BUILD.md`，记录 APK 首包、wgt 热更新、生产地址、版本号、强制更新和发布边界。
- 新增 `APP_RELEASE_CHECKLIST.md`，固化 APK 首包和 wgt 热更新发布检查项。
- `.gitignore` 忽略 `certs/`，避免 Android keystore 和签名密码相关文件误提交。
- 前端关于页版本展示改为读取统一配置；后端 `/api/common/config` 返回当前服务版本，便于 App 同步展示。
